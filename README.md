# Google BigQuery Retail Sales Analysis Project

## Problem Statement

Retail businesses generate large volumes of transactional data daily. However, raw data alone does not provide meaningful insights for decision-making. Businesses need to analyze sales transactions to understand revenue performance, customer demographics, product category trends, and spending patterns.

This project uses Google BigQuery and SQL to analyze a Retail Sales Dataset and answer key business questions through data filtering, aggregation, categorization, and reporting techniques.

---

## Aim of the Project

The aim of this project was to develop practical SQL skills using Google BigQuery while performing data analysis on a retail sales dataset.

The project focused on extracting valuable business insights from transactional data and applying SQL techniques commonly used by Data Analysts in real-world environments.

---

## Objectives

To achieve the aim of the project, the following steps were performed:

### 1. Data Exploration

* Reviewed the structure of the Retail Sales Dataset.
* Identified relevant columns required for analysis.

### 2. Data Filtering

* Filtered transactions that occurred during specific periods.
* Applied conditional statements to isolate relevant records.

### 3. Revenue Analysis

* Calculated total revenue generated from all transactions.
* Identified transactions exceeding average sales values.

### 4. Product Category Analysis

* Retrieved unique product categories.
* Calculated total quantities sold per category.
* Identified categories generating significant revenue.

### 5. Customer Segmentation

* Classified customers into age groups:

  * Youth
  * Adult
  * Senior
* Analyzed customer demographics using SQL CASE statements.

### 6. Transaction Classification

* Categorized products based on unit price:

  * Cheap
  * Moderate
  * Expensive
* Classified customer spending levels based on transaction values.

### 7. Business Reporting

* Applied GROUP BY and HAVING clauses to summarize data.
* Generated insights that support business decision-making.

---

## Summary of Results

The analysis successfully demonstrated how SQL can be used to transform raw retail transaction data into meaningful business insights.

Key outcomes included:

* Identification of total revenue generated from sales transactions.
* Analysis of product category performance.
* Customer segmentation based on age groups.
* Identification of high-value transactions.
* Classification of products according to pricing categories.
* Evaluation of customer spending behaviour.
* Creation of business-focused reports using SQL aggregation techniques.

The project strengthened practical knowledge of data querying, data transformation, and analytical problem-solving using Google BigQuery.
